Layouts that wrap

That’s a wrap

Test these layouts in narrow, medium and wide views!

Give an element a class="wrap" to wrap its children elements across rows. To do this, the children elements will have to be given a base width by using the flex-basis property (make sure you duplicate this with a -webkit- prefix for poor Safari) – or you can simply use left and right padding or margin. If you do this, the .wrap items will render a fluid flow of content across multiple rows, without following any particular grid structure.

The last row, even if containing fewer elements than other rows, will fill the available width of the parent.

Layouts that wrap

Defined wrapping layouts

Test these out in medium and wide views!

These are columnar layouts that repeat across rows. They do not require the use of the wrap class (although they can be used together – see the last example on this page), but rather one or more classes defining the column structure. The classes will include a basic division of the parent width, sometimes a multiple of those divisions, and sometimes also a direction. Examples:

<article class="fourths"></article>
<article class="two fourths"></article>
<section class="two thirds right"></article>
<section class="six sevenths left"></article>

Check these out in narrow views!

Sometimes you want a single element to align itself left or right, with all elements after (or sometimes before) to align themselves on the opposite side. These are the only layouts that use float instead of flex, and are simple to accomplish in Sessile. Just give the parent element a class of .one, plus whatever division you would like. Examples:

<section class="one third left"></section>

Layouts that don’t wrap

Row, row, row

Test these out in narrow views!

Give an element a class="row" to render its child elements horizontally within a single row.

With that single class, child elements will each take up the same width, no matter how many they are. That’s right – as many as you want! (But try not to break the browser, okay?)

Give the parent element padding-left and/or padding-right to add space on either side, or both.

Defined row layouts

Some examples in lists

Test these out in narrow views!

Certain defined layouts are possible used in conjunction with the row class. Because all items are within a single row, things are quite flexible: there’s no need to specify particular widths or divisions. Examples:

<ul class="row two left"></ul>

In this layout, no matter how many lis are contained, the first will have twice the width of the others: so, if there are three lis, the first will be 50% width (the other two being 25% each); if four lis, it will be 40% width (the other three being 20% each); and so on.

Navigation

In addition to the columnar and wrapped layouts described above, Sessile Lite offers a few navigation solutions – since navigation layout is also, well, layout.

Examples used in this page: top fixed nav, bottom fixed nav, universal nav, global nav.

Global and universal navigation

At times it is desirable to have a navigation element natively positioned at the bottom of the content for handheld-width views, but positioned at top for tablet and desktop-width views. Sessile Lite offers an easy solution: nest a nav element within a parent with a class group of .global.nav, and the nav element will be moved to the top, underneath the top header, for the medium and wide views. Magic!

Apply a class group of .universal.nav to the parent element, and the nav element will be moved to the top as before, but this time above the header! Or, how about this: two nav elements, both nested inside a parent with .universal.global.nav – both are moved to the top of the page, the first above the header, the second below the header.

Awesome.

For universal or local navigation, you may use either a ul link list, or, as is sometimes done (and is done in the univeral navigation on this sample page), with span elements nested directly inside of the nav.

Local navigation

You can also put a nav element as the last child of an element with a class group of local nav, and it will rise to the top of that parent element in medium view, and participate in wide view layouts. A local navigation, unlike global and universal navigation, will not display as a row by default.

Fixed navigation

This one is easy. Place a nav element outside of your container div, directly inside the body element, and that nav will stretch the width of the viewport. Place it just after the opening body tag, and the fixed navigation will be at top; just before the closing body tag, and it will be at the bottom.

To fix these navigations, add a class of fixed to the parent – in this case, the body element.* Violá!

*By default, a bottom fixed navigation remains fixed in narrow view, while a top fixed navigation does not. Too many different top navigation solutions available to the developer for Sessile to start dictating things there. Apply your own solution.

The navigation code on this page:

<body class="fixed nav top bottom">
    <nav></nav><!-- this navigation stretches at top, position fixed in medium and large views -->
    <div class="universal global nav">
        ... <!-- header and main -->
        <nav></nav><!-- this navigation positioned above the page header -->
        <nav></nav><!-- this navigation positioned below the page header -->
    </div>
    <nav></nav><!-- this navigation stretches at bottom, position fixed all views -->
</body>

Here’s an explanation of Life, the Universe, and Everything.*

Okay, so maybe not. But here is an explanation of Sessile Lite:

Note: this explanation will appear at the bottom of each screen in this sample page!

Sessile Lite may be used as a standalone library, or as part of the larger Sessile front-end framework (soon to come!).

Three views are provided: narrow, medium and wide. They are contained within three different CSS files, rather than a single file. The purpose behind this is to allow the developer to use custom media queries when referencing the CSS, applying breakpoint widths that are appropriate to each unique project. Whoa, concept, huh?

Your CSS links might look something like this:

<link rel="stylesheet" href="narrow.css">
<link rel="stylesheet" href="medium.css" media="screen and (min-width: 40em)">
<link rel="stylesheet" href="wide.css" media="screen and (min-width: 60em)">

… though again: fully customizable, you can change these to whatever you project requires. If, for instance, you wanted to have the medium.css and wide.css rules apply at the same view, simply give them the same min-width value.

Use all three, or just the ones that are useful for your project. Customize everything by linking your own theme.css (or whatever) file after these three. And so on.

Minified versions of each CSS file are provided as well. Minification was accomplished at...

The way the layouts work

Specifying widths in the classes is generally unnecessary: by default, layouts applied to sub-module elements render in narrow view, module elements (header, footer, aside, or any direct child of article) in medium view, and architectural elements (main, body > div) in wide view. Remember that breakpoints for the three views are completely customizable, so this is much more permissive than it may seem at first flush.

Layout renders do not by default persist into wider views. To make a narrow layout continue into medium view, add a class of persistent (example: <ul class="row persistent">). Layouts in medium view automatically persist into wide view.

Warning: filling elements with content that forces them to grow may cause layouts to break at wider views. This can happen at times when persisting a narrow layout while simultaneously implementing a medium layout. It is recommended to truncate long lines using CSS or JavaScript.

Another warning: Since adding margins to an element can increase its calculated size, adding left and/or right margins to layout items can break these layouts. Until CSS provides some version of box-sizing: margin-box or similar, Sessile Lite makes no attempt to solve this problem, leaving it for the larger Sessile framework to fix. Coming soon! In the meantime, padding and borders, and top and/or bottom margins, are fine to use and should not break anything.

Yet another warning: Sessile Lite makes heavy use of CSS3 features such as nth-child selectors, flex properties, and calc() measurements, and is therefore not particularly well-suited to supporting legacy browsers on its own. However, the potential exists to use Sessile Lite along with a feature detecter such as Modernizr and a polyfill such as Flexie (or your own). As of the current beta version of Sessile Lite, this has not yet been tested.

Adding classes to your markup

Rather than needing to add a class or more to each and every layout item, simply add those classes to the parent element, and the layout will be applied to its direct children elements. Plenty of examples are given in this sample document. Browse through the links!

Note: In this sample.html document, the divs nested directly inside main are completely unnecessary for the layouts; they are there only to provide id targets for the navigation!
